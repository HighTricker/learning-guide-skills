# 产品需求文档：学习skills组合

## 一句话定义
一个skills包，用户安装后可以直接利用Claude辅助学习各类知识，这个包可以根据你的学习目标帮助你判断学习内容，学习顺序，可以判断你的理解是否正确，可以判断你对于知识的应用的理解是否正确，可以结合你的实际应用方向给你出题以检验你的学习效果。

## 用户画像
多人使用，最简单的安装方式，一行命令即可

## 使用场景
用户在项目目录安装skill包，写好Claude.md文件后开始学习，在指定的文件中写好自己的理解，应用，大模型进行评判，之后大模型出题，用户回答，大模型再次评判，帮助用户加深理解。

## 页面结构
无页面结构，全部是纯md格式文件。

#### 数据逻辑
用户自己下载好学习资料，推荐为md格式的文本。
大模型自己读取学习资料并调用skills进行对应操作。
操作逻辑如下：
1. 读取用户的学习目标和应用目标。
2. 读取学习资料。
3. 对比两个读取的结果，明确需要学习的内容和学习的顺序，将这个写为一个md文件，作为学习计划。
4. 调用skill创建学习笔记，笔记内容包括学习章节名称，用户的理解，大模型对理解的批注，用户的应用，大模型对应用的批注，大模型的出题，用户对题目的回答，大模型对用户答案的批注。

其中，大模型出题的时候需要引用对用户的理解的批注和对用户应用的批注，以便实现查漏补缺的效果。


## 功能清单
### MVP（第一版）
- [ ] 用户注入skill，大模型创建一个文档，文档中包括用户的学习目标和用户的已有应用内容。用户自己写完所有文档或者和大模型交互，大模型完成该文档。这个文档是学习的出发点，需要什么学习什么，渐进式披露知识。
- [ ] 学习计划skill，对比学习资料文件和学习目标文件，明确需要学习的内容和学习的顺序，将其开发为学习计划文档。
- [ ] 理解批注skill，对比学习资料和用户自己写的理解，在指定位置写出批注，明确哪些正确，哪些错误。
- [ ] 应用批注skill，对比学习资料，用户的应用情况，在指定位置写出批注，明确哪些正确，哪些错误，有什么用户没想到的地方。
- [ ] 出题skill，根据对用户的理解的批注和对用户应用的批注重点出题，查漏补缺。在用户回答后，进行批注，明确哪些正确，哪些错误，有什么用户没想到的地方。

## 成功标准
用户一行命令能安装skill包，并能在学习目录下实现以上功能。

# 功能详述
至少分为三个 skills：
1. 第一个负责获得学习材料和用户的理解，之后根据学习材料去审核、评判用户的理解是否到位，是否有遗漏补缺的地方
2. 第二个 skill 负责获取用户关于学习目标以及自己实践项目的基本信息。根据用户在本节学习内容之后写的应用内容，结合实际项目的资料与其区别，做一个联合评审，评估用户编写的情况是否符合实际。
3. 第三个 skill 负责出题：根据用户设定的角色范围去出相应的题目，并在用户答题之后进行评审。
4. 一个skill负责生成学习笔记文件。
