# 应用批注

## 何时使用
**同时满足**（V1.1 标识符判定）：
- 存在 `学习笔记.md`
- 某章节「我的应用」字段下**不含** `[请在此处写`
- 同章节「大模型批注（应用）」字段下**含** `[AUTO]`

## 操作流程

### Step 1 — 定位章节
- `Grep '^## 第' 学习笔记.md -n` 拿章节起始行号
- 找到用户最近写完「我的应用」、批注还含 `[AUTO]` 的章节

### Step 2 — 状态检查（按 §4.6）
```bash
grep -A 3 "### 我的应用" 学习笔记.md | grep -v "\[请在此处写"
grep -A 3 "### 大模型批注（应用）" 学习笔记.md | grep -q "\[AUTO\]"
```

异常时按 §4.3 处理。

### Step 3 — 读取三方上下文
- 该章节的「我的应用」内容（按大文件读取规范）
- `学习目标.md` 中的「已有背景」「当前项目 / 应用场景」段
- `学习计划.md` 找该章节对应的资料，Read 该资料章节正文

### Step 4 — 三方对比生成批注（V1.2 emoji-only）
**这是本 skill 与 [[理解批注]] 的核心区别**，必须三方对比：

| 对比维度 | 内容 |
|---|---|
| 正确性 | 用户应用方案 ↔ 学习资料的标准做法 |
| 结合度 | 用户应用方案 ↔ 用户实际项目背景（从 `学习目标.md` 读） |
| 遗漏点 | 资料里有但用户没用上的（💡） |

用 emoji + 简短描述（**严禁包 HTML span 标签**）：
- ✅ 做法正确 / 与项目契合
- ⚠️ 做法可行但有改进空间
- ❌ 做法不当（说明原因 + 正确做法）
- 💡 资料里推荐但用户没用的（明确说"针对你的 [项目名/技术栈] 建议补 X"）

### Step 5 — 写入批注（替换占位整行）
用 Edit 精确替换：

- `old_string` = `> 🤖 **[AUTO]** Claude 会在此处自动填批注，请勿编辑`
- `new_string` = Step 4 生成的批注

**其他字段一律不动**。

### Step 6 — 告知用户
输出："✅ 已批注第 N 章「我的应用」。理解和应用都齐备了。需要做题检验你的掌握情况吗？说一声我就出题。"

## 契约
- **读入**：`学习笔记.md`、`学习目标.md`、`学习计划.md`、`资料/*.md`
- **写出**：`学习笔记.md`（仅对应章节「大模型批注（应用）」字段）

## 大文件读取（强制）
同 [[理解批注]]，先 Grep 定位再 Read 局部。

## 注意事项
- **必须读 学习目标.md**：每次都要引用用户档案中的项目背景，这是本 skill 与 [[理解批注]] 的关键差异
- **不假设用户没写的部分**：用户只写了一段就只批一段，不要替用户脑补"你可能还想做 X"
- **结合实际项目要具体**：批注里要明确说"针对你的 [项目名/技术栈]，这个做法 [合适/不合适，原因是 xxx]"，而不是泛泛而谈
- **严禁包 HTML span 标签**：V1.2 已回退到 emoji-only
- **找不到 学习目标.md** → 提示"请先 /学习信息注入 建立档案"，退出
