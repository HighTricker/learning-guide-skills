# 理解批注

## 何时使用
**同时满足**（V1.1 状态检查改用标识符）：
- 存在 `学习笔记.md`
- 某章节「我的理解」字段下**不含** `[请在此处写`（= 用户已填）
- 同章节「大模型批注（理解）」字段下**含** `[AUTO]`（= 待批注）

不满足时不要触发。

## 操作流程

### Step 1 — 定位章节
- `Grep '^## 第' 学习笔记.md -n` 拿所有章节起始行号
- 找到用户最近写完的那一章（用户消息会暗示章节号；不确定就用 Grep `-A 3` 检查所有章节的「我的理解」和「大模型批注（理解）」字段，找到"我的理解已填 + 批注未填"的章节）

### Step 2 — 状态检查（按 §4.6 标识符 Grep 速查）
```bash
# 用户字段必须已填
grep -A 3 "### 我的理解" 学习笔记.md | grep -v "\[请在此处写"
# 大模型字段必须待办
grep -A 3 "### 大模型批注（理解）" 学习笔记.md | grep -q "\[AUTO\]"
```

如果用户字段还含 `[请在此处写` → 按 §4.3 提示"你还没写第 N 章「我的理解」"，退出。
如果大模型字段不含 `[AUTO]` → 该章已批注过，退出。

### Step 3 — 读取上下文（仅当前章节）
按下方"大文件读取强制规范"：
- Read `学习笔记.md` 时用 `offset` 和 `limit` 只读当前章节范围
- Read `学习计划.md` 找该章节对应的学习资料文件
- 用 Grep 在资料文件里定位该章节，Read 该章节正文（不读全文）

### Step 4 — 对比生成批注（V1.2 emoji-only）
对照学习资料，逐点对比用户的理解，用 emoji + 简短描述输出。**严禁包 HTML span 标签**：

- ✅ 正确 / 到位（绿色 emoji）
- ⚠️ 部分正确 / 理解不完整（黄色 emoji）
- ❌ 错误（红色 emoji，说明错在哪、正确是什么）
- 💡 拓展提示 / 没想到的关键点（黄色 emoji）

示例输出：
```
✅ 无状态的理解到位
❌ HTTP 与 TCP 混淆 —— HTTP 是应用层，TCP 是传输层
⚠️ "请求-响应"理解不完整：漏了 HTTP/2 多路复用
💡 没提到无状态如何配合 Cookie/Session 模拟会话
```

每条一行，避免长段落。

### Step 5 — 写入批注（V1.1 替换占位整行）
用 Edit 精确替换该章节「大模型批注（理解）」字段下的整行占位：

- `old_string` = `> 🤖 **[AUTO]** Claude 会在此处自动填批注，请勿编辑`
- `new_string` = Step 4 生成的批注（不再含 `[AUTO]`）

**其他字段一律不动**。

### Step 6 — 告知用户
输出："✅ 已批注第 N 章「我的理解」。你可以继续在「我的应用」字段下方写你的应用方案。"

## 契约
- **读入**：`学习笔记.md`（定位章节）、`学习计划.md`（找资料映射）、`资料/*.md`（对比）
- **写出**：`学习笔记.md`（仅修改对应章节的「大模型批注（理解）」字段，其他字段一律不动）

## 大文件读取（强制）
按技术细则.md §4.1：
1. 必须先 `Grep '^## 第' 学习笔记.md -n` 定位
2. Read 时用 `offset` 和 `limit` 只读目标章节范围
3. 禁止全文 Read（除非笔记 <200 行）

## 注意事项
- **只批注一章**：一次只处理用户最近写完的那一章，不要批量处理多章
- **不重写用户内容**：「我的理解」字段下用户写的内容**严禁修改**
- **批注简短**：每条一行，长解释拆成多条 💡
- **严禁包 HTML span 标签**：V1.2 已回退到 emoji-only，包 span 会暴露 raw HTML 干扰阅读
- **找不到对应资料章节** → 提示用户检查 学习计划.md 的章节映射，本 skill 退出
- **用户错位输入识别**：如果发现「大模型批注（理解）」下不含 `[AUTO]` 但内容也不像 Claude 生成的批注（无 HTML span），按 §4.3 提示用户写错位置
