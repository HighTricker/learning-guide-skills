# 学习信息注入

## 何时使用
- 用户首次进入学习目录，还没有 `学习目标.md` 文件
- 用户明确要求"开始学习 / 制定学习目标"
- 已有档案但用户要求"更新学习目标"或"补充应用方向"

## 操作流程
1. 先用 Glob 检查当前目录是否已有 `学习目标.md`
   - 已存在 → 询问用户是覆盖还是补充
   - 不存在 → 进入下一步
2. 用 Write 创建 `学习目标.md`，按下方模板生成骨架（所有字段先留占位符 `[待用户填写]`）
3. 渐进式追问用户（**一次只问 1-2 个问题**，不要一次性灌输全部）：
   - 学习目标是什么？想达到什么水平？
   - 当前已有的应用场景 / 项目背景？
   - 期望的学习周期？
4. 每收到用户回答，用 Edit 把对应字段从占位符替换为答案
5. 全部字段填完后，告诉用户："档案已建好。把学习资料放进当前目录（推荐放 `资料/` 目录下），然后调用 `/学习计划制定` 进入下一步。"

## 输出模板（学习目标.md）

````markdown
# 学习目标

## 学习目标
- 主目标：[待用户填写]
- 子目标：
  - [ ] [待用户填写]

## 已有背景
- 相关经验：[待用户填写]
- 当前项目 / 应用场景：[待用户填写]

## 学习偏好
- 期望周期：[待用户填写]
- 偏好的学习方式：[待用户填写，可选：看资料 / 做项目 / 出题练习]
````

## 契约
- **读入**：无（首次创建）
- **写出**：`学习目标.md`（供 [[学习计划制定]] 读取）

## 注意事项
- **不要替用户写学习目标**：必须通过对话获取用户的真实回答，禁止脑补
- **档案文件名固定为** `学习目标.md`，不要改名 —— 其他 skill 会按这个名字读
- **目标必须可衡量**：用户回答"学 Python" 这种粒度时，追问"想用 Python 做什么具体事情？比如能独立写一个爬虫？做数据分析？"
- **应用场景必须具体**：避免"做一个项目"这种空话，追问到具体技术栈和场景
