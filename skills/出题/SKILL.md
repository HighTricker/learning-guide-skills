# 出题

本 skill 包含两个分支：**出题分支**（生成新题）和**评判分支**（评判已答题目）。Claude 根据笔记当前状态判断走哪个分支，两个分支互斥。

## 何时使用

### 出题分支触发条件（V1.1 标识符判定）
- 某章节「大模型批注（理解）」和「大模型批注（应用）」均**不含** `[AUTO]`（= 已批注）
- 同章节「出题与答题」字段下**含** `[AUTO]`（= 待出题）
- 用户表达"想做题 / 检验一下 / 出题 / 考考我"等意图

### 评判分支触发条件（V1.1）
- 某章节「出题与答题」下已有题目（即字段不含 `[AUTO]`）
- 题目下「我的答案 N」字段**不含** `[请在此处写`（= 用户已答）
- 同题目「大模型批注 N」字段**含** `[AUTO]`（= 待评判）

## 操作流程

### 出题分支

#### Step 1 — 定位章节
按下方"大文件读取强制规范"定位目标章节。

#### Step 2 — 状态检查（按 §4.6）
```bash
grep -A 3 "### 大模型批注（理解）" 学习笔记.md | grep -v "\[AUTO\]"
grep -A 3 "### 大模型批注（应用）" 学习笔记.md | grep -v "\[AUTO\]"
grep -A 3 "### 出题与答题" 学习笔记.md | grep -q "\[AUTO\]"
```

异常时按 §4.3 处理。

#### Step 3 — 提取查漏补缺重点
Read 两个「大模型批注」字段，提取所有 ⚠️ ❌ 💡 条目作为出题素材：
- ❌ → 概念错误，出题验证是否纠正
- ⚠️ → 理解不完整，出题考完整性
- 💡 → 没想到的点，出题考补漏

#### Step 4 — 自适应选型 3-5 道题
按重点类型自适应选题型：

| 重点类型 | 推荐题型 |
|---|---|
| 概念模糊 / 名词混淆 | 选择题 |
| 理解不全 / 流程顺序 | 简答题 |
| 应用脱节 / 实战场景 | 应用题（必须结合用户项目） |

#### Step 5 — 写入题目骨架（V1.1 新占位）
用 Edit 精确替换该章节「出题与答题」下的整行占位：

- `old_string` = `> 🤖 **[AUTO]** Claude 会在此处出题，你在每道题下方写答案`
- `new_string` = 以下结构（每题留出答案 + 批注占位）：

````markdown
#### 题目 1（针对 xxx 批注）
[题目内容]

##### 我的答案 1
> ✍️ **[请在此处写你的答案]** —— 写完保存后告诉 Claude

##### 大模型批注 1
> 🤖 **[AUTO]** 待评判

#### 题目 2（针对 xxx 批注）
（同上结构）
````

#### Step 6 — 告知用户
输出："✅ 已为第 N 章出 N 道题，覆盖 [理解+应用的查漏点]。在每道题的「我的答案 N」**引用块下方**（不是引用块里！）写你的答案，写完告诉我。"

---

### 评判分支

#### Step 1 — 定位章节 + 题目
找到所有"答案不含 `[请在此处写` + 批注含 `[AUTO]`" 的题目。

#### Step 2 — 评判每道题（V1.2 emoji-only）
Read 题目 + 答案，对照学习资料判断正确性，用 emoji + 简短描述（**严禁包 HTML span 标签**）：

- ✅ 完全正确：...
- ⚠️ 部分正确：缺 ...
- ❌ 错误：... 正确是 ...
- 💡 答对了但还能更进一步：...

#### Step 3 — 写入批注（替换整行占位）
对每道题，用 Edit 替换：

- `old_string` = `> 🤖 **[AUTO]** 待评判`
- `new_string` = 该题的评判批注

#### Step 4 — 告知用户
输出："✅ 第 N 章答题已评判，共 N 道：[✅ X / ⚠️ Y / ❌ Z]。学习总结即将自动生成。"

**不再自己写总结**，交给 [[学习总结]] skill。

## 契约
- **读入**：`学习笔记.md`（批注 + 答案）、`学习目标.md`（项目背景）、`学习计划.md`（资料映射）、`资料/*.md`
- **写出**：`学习笔记.md`（仅对应章节「出题与答题」字段）

## 大文件读取（强制）
同 [[理解批注]]，先 Grep 定位再 Read 局部。

## 注意事项
- **题目必须标出题理由**：明确指向哪条批注（如"针对 ❌ 状态码混淆"），让用户知道为什么考这个
- **应用题必须结合实际项目**：从 `学习目标.md` 读项目背景，让题目贴近用户场景
- **评判分支不要追加新题**：用户答完只评判，禁止画蛇添足出新题
- **评判完不写总结**：明确把控制权交回主对话，由 [[学习总结]] skill 接力
- **3-5 道题为限**：宁少勿多，每道题都要有针对性
- **严禁包 HTML span 标签**：V1.2 已回退到 emoji-only
- **如果 token 消耗大**：开发后观察实际表现，V2 考虑加 `context: fork`
