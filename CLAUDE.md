# 学习指导 Skills 包

## 项目定位
一个 Claude Code skills 包。把日常学习的完整路径拆解为 6 个 skill：注入学习目标 → 制定学习计划 → 写理解/应用 → 大模型批注 → 大模型出题 → 用户答题 → 大模型评判 → 学习总结。用户一行命令安装后，通过与 Claude 协作完成学习任务。

## 当前阶段
**V1.1 已完成**：MVP 6 个 skill 全部就绪 + 部署可用 + 端到端测试通过。
**下一步**：根据用户使用反馈做迭代。开放问题（V2 路线）见底部。
**禁止跳阶段**：不要修改 `技术细则.md` §1-6 的设计与"已对齐决策"表。

## 关键文件
- `PRD.md` —— 产品需求文档
- `技术细则.md` —— **设计 ground truth**：共享契约 + 6 个 skill 完整设计稿
- `开发计划.md` —— 开发任务清单（MVP 阶段）
- `CLAUDE.md`（本文件）—— 项目协作规范
- `skills/` —— 6 个 skill 源码
- `install.sh` / `README.md` —— 安装脚本与用户文档

## 协作模式
- **每次只推进一个技术决策**，不要一次性抛出大量问题让用户疲于决策。
- 多选项的决策用 `AskUserQuestion` 让用户选，单一明确的问题直接问。
- "参考资料" = `PRD.md` + `技术细则.md`。开发 skill 必须严格遵循其中的约束。
- 用户的口头描述可能有语音输入误识（如 "Cloud" → "CLAUDE"），按上下文合理纠正后告知。

## Skill 开发硬约束（强制遵守）
1. **设计稿在 技术细则.md §5**：每个 skill 的 frontmatter 和 body 概要已经定好，按稿子创建实际文件，不要重新设计。
2. **共享契约在 技术细则.md §1-4**：协作方案 / 文件契约 / 笔记 schema / 公共操作规范，所有 skill 必须遵守。改一处 6 个 skill 都得改。
3. **500 行法则**：SKILL.md 必须 <500 行，超过拆 `templates/` `examples/` 子文件。
4. **契约式引用三要素**：body 中引用其他文件必须写齐 触发条件 + 路径 + 内容预期（参考 技术细则.md §4.2）。
5. **大文件读取规范**：操作 `学习笔记.md` 必须先 Grep 定位章节再 Read 局部，禁止全文 Read。
6. **权限最小化**：`allowed-tools` 按需声明，参考型 skill 通常只需 `Edit` 不需要 `Write`。
7. **状态判断用标识符**（V1.1）：判定字段已填靠 Grep `[请在此处写` / `[AUTO]`，不再判断"字段为空"。
8. **批注用 emoji-only**（V1.2）：✅⚠️❌💡，**严禁包 HTML span 标签**。emoji 自带颜色（绿/黄/红/黄），所有渲染器一致显示。
9. **命名规范**：见 技术细则.md §6。
